Keys的作用

对字节点进行递归

在默认条件下，当递归遍历两个DOM节点的子元素，如果遇到不同的元素时，会对其进行更改。如果在子元素的末端添加新元素，变化的成本就比较低。React 文档 介绍了为什么要使用key。

在末端插入，react会从第一个开始匹配，发现两个元素一样。跳到下一个元素进行匹配。直到最后，插入第三个元素

<ul>
  <li>first</li>
  <li>second</li>
</ul>

<ul>
  <li>first</li>
  <li>second</li>
  <li>third</li>
</ul>

在列表的头部插入，从第一个元素开始，发现不匹配，进行修改。但实际中，两个元素是 相同的，但位置发生了改变。

<ul>
  <li>Duke</li>
  <li>Villanova</li>
</ul>

<ul>
  <li>Connecticut</li>
  <li>Duke</li>
  <li>Villanova</li>
</ul>

key 解决了上面的问题

<ul>
  <li key="2015">Duke</li>
  <li key="2016">Villanova</li>
</ul>

<ul>
  <li key="2014">Connecticut</li>
  <li key="2015">Duke</li>
  <li key="2016">Villanova</li>
</ul>

当子元素拥有key时，react使用key来匹配原有树上的子元素已经新的树上的子元素。让其变得高效.

设置key的方案很简单，如下所示
<li key={item.id}>{item.name}</li>

用props来传递数据

组件中的props是只读，不能重新赋值

import React from 'react'
import ReactDOM from 'react-dom'


function Hello(props){
return <div>Hello World --- {props.name} age: {props.age}</div>
}

const dog = {
    name: 'GIGI',
    age: 3
}


ReactDOM.render(<Hello name={dog.name} age={dog.age}></Hello>,document.getElementById('app')) 
//更简便的写法
ReactDOM.render(<Hello {...dog}></Hello>,document.getElementById('app'))

基于class关键字创建组件

  render(){
    return <div>class 创建的组件</div>
  }
}
class 和 function 创建组件的区别
  • 用class创建的组件，有自己的私有数据和生命周期
  • 用function创建的组件，没有私有数据和生命周期；效率会更高一些，但没有state属性
  • 有无state属性

在class创建的组件中，this.state中的数据是可读可写的

class Movie extends React.Component {

    constructor(){
        super();
        this.state = {
            msg: 'state message'
        }
    }
    render(){
        this.state.msg = 'new state message'
        return <div>{this.state.msg}</div>
    }
}

组件中的props和state/data的区别

  • props 中的数据都是外部传递的
  • state/data 中的数据，都是组件私有的；(可以通过 Ajax 获取回来的数据，一般都是私有数据)
  • props 中的数据是可读的
  • state/data 中的数据，都是可读可写的;
